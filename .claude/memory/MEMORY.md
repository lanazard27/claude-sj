# 세션 간 상태 기억
> CLAUDE.md에 있는 규칙은 여기에 적지 않는다. 세션 넘어가야 하는 "상태"만 기록.

# 사용자 선호
- **테스트/점검 작업 시 중간 확인 없이 결과만 보고** — 완료 전에 물어보지 말 것
- **검색/조사할 때 알림 없이 진행** — 최종 결과만 보고
- **테스트 중 알림/차단 발생 시 메모리 업데이트 후 자체 해결하고 계속 진행** — 사용자 개입 없이 자동 복구
- **테스트 전체 권한 위임** — 모든 결정은 sj가 알아서, 결과만 보고

# 일반 교훈
- **컴팩트 후에는 반드시 기존 작업 상태 검증부터** — "안 날아갔어" 단정 금지
- **사용자가 "날아갔다"고 하면 바로 믿고 검증**
- **패치 2번 이어지면 근본 원인(재시작 필요) 의심**

# 에이전트 운영 교훈
- **병렬 스폰 3개+면 code-writer가 파일 누락** — 보고서만 쓰고 cat > 명령 생략. 항상 파일 존재 확인 후 재스폰
- **비표준 라이브러리(sql.js 등)는 API 사용법을 프롬프트에 포함** — 타입 선언을 에이전트가 임의로 만들면 잘못된 시그니처 → 런타임 버그
- **에이전트 프롬프트에 라이브러리 사용 패턴 예시 포함** — run(sql, params?), exec(sql, params?) 같은 핵심 API 서명 직접 기재

# 도구 선호
- **MCP 웹검색(mcp__web-search-prime) 사용 금지** — 토큰 빨리 소진됨. 일반 WebSearch만 사용
