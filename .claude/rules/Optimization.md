# 시스템 최적화 규칙

> "최적화해", "정리해", "압축해" 등 최적화 요청 시 반드시 따를 것.

## 기본 원칙

### 1. 기능 무손실 (최우선)
- 압축 = 표현 방식의 간소화, 기능의 삭제가 아님
- 압축 전후에 기능 체크리스트를 반드시 대조
- "이건 안 쓸 것 같은데?" → 삭제 금지, 사용자에게 먼저 확인

### 2. 최적화 전 필수 절차
1. **기능 인벤토리 작성** — 현재 파일의 모든 기능/규칙을 항목화
2. **압축 실행** — 표현만 간소화
3. **기능 대조 검증** — 인벤토리와 압축 결과 1:1 대조, 누락 없는지 확인
4. **누락 발견 시** — 즉시 복구, 사용자에게 보고

### 3. 위험 패턴 (피해야 할 것)
| 패턴 | 문제 | 대안 |
|------|------|------|
| 예시/코드블록 제거 후 핵심 동작 설명 누락 | "어떻게"가 사라짐 | 동작을 텍스트로 먼저 기록 후 예시 제거 |
| 표 → 한 줄 요약 시 출처 정보 유실 | 데이터 출처가 섞이거나 누락 | 각 항목별 출처를 개별 명시 |
| 참조형으로 위임 시 대상 파일에 내용 없음 | dead reference | 위임 전 대상 파일에 실제 내용 있는지 확인 |
| 여러 출처를 하나로 합칠 때 일부 누락 | A+B → A만 남음 | 합치기 전 각 출처의 기여 항목 명시 |

### 4. Source of Trust 원칙
- 중복 제거 시 **어느 파일이 진실**인지 명확히 결정
- 진실 파일에 상세 내용이 **실제로 존재하는지** 확인 후에만 다른 파일에서 제거
- 참조만 남길 때: `[파일명] 참조` 형태로 명시

### 5. 검증 체크리스트
최적화 완료 후 반드시:
- [ ] 압축 전 기능 항목 수 == 압축 후 기능 항목 수
- [ ] 참조 대상 파일에 실제 내용 존재
- [ ] 데이터 출처 정확성 (A+B가 A만으로 축소되지 않았는지)
- [ ] 새로운 파일에서 해당 규칙 읽었을 때 동일하게 행동 가능

### 6. 참조형 분리 (큰 블록 이동)
- **대상**: 템플릿, 코드블록, 다이어그램 등 10줄+ 블록이면서 매번 로드할 필요 없는 것
- **방법**: 별도 파일로 이동 → 원래 위치에 `[파일경로] 참조` 한 줄만 남김
- **조건**: 분리 후 원본 파일에서 같은 기능 수행 가능해야 함
- **예시**: 세션 저장 형식 → `.claude/sessions/template.md` 분리
