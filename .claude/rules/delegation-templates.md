# 에이전트 위임 프롬프트 템플릿

> sj가 에이전트를 스폰할 때 반드시 이 템플릿을 사용한다.
> **원칙: sj는 "요약"하지 않고 "복사"한다.** 이전 에이전트의 출력은 그대로 전달.

## 공통 규칙 (모든 템플릿에 적용)

1. **Verbatim 복사** — architect/explorer 출력을 sj가 재작성하지 않음. 마크다운 블록 그대로 전달
2. **결정사항 명시** — sj가 미리 결정한 기술적 선택(버전, DB, 설정값)을 포함. 에이전트가 결정할 여지 없앰
3. **범위 명시** — 수정 가능/금지 파일 목록 항상 포함
4. **제약 공통** — "한국어, 질문 금지, 즉시 실행, Bash cat > 사용"

## 출력 수준 요구사항 (매우 중요)
반드시 아래 수준으로 설계할 것:
- 함수 시그니처: 파라미터명 + 타입 + 반환 타입 포함
- 타입 정의: 실제 TypeScript interface/type 코드로 작성
- 핵심 로직: 의사코드로 단계별 기술 (writer가 번역만 하면 되는 수준)
- 에러 처리: 모든 에러 케이스를 표로 매핑
- DB 쿼리: Prisma 메서드명까지 명시 (findUnique, upsert 등)
- 기존 코드 참조: "이 파일:이 함수를 참고" 수준으로 구체화
- import 경로: 절대 경로로 명시

- 하나의 최적 접근법에 확신 있게 커밋

- 컴포넌트 수준 설계 금지 — 반드시 코드 수준(시그니처 + 의사코드)까지

## 구현 지시사항 (architect 설계 기반)
1. **인터페이스 계약서에 정의된 타입/시그니처를 그대로 구현** — 임의 변경 금지
2. **의사코드를 실제 코드로 번역** — 로직 순서를 임의로 변경하지 말 것
3. **기존 코드 참조된 파일은 반드시 Read로 읽고 동일 패턴 적용**
4. **에러 시나리오 테이블의 모든 케이스를 구현에 반영**
5. **구현 제약사항(금지/강제)을 엄격히 준수**
