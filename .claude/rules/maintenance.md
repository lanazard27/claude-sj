# 메모리 관리, 하네스 유지보수, 피드백 규칙

## 메모리 관리

### 기본 원칙
- **모든 프로젝트 메모리는 별도 디렉토리에서 관리** — 절대 홈 MEMORY.md에 프로젝트 내용을 넣지 말 것
- 홈 MEMORY.md에는 **사용자 선호/규칙만** 기록
- 프로젝트별 내용은 해당 프로젝트 디렉토리의 memory에 기록
- 분야별 선호 패턴, 스타일 발견 시 MEMORY.md에 기록
- 실패 이력, 반복되는 수정 사항을 기록

### 자동 최적화 (세션 종료 시)
- **MEMORY.md**: 150줄 초과 시 최근 120줄만 유지 (200줄 제한에서 여유 확보)
- **스냅샷**: 7일 이상 된 세션 스냅샷 자동 정리
- **experience 카테고리**: 5개 초과 시 session-start에서 압축 권고 표시
- **통계**: 30일 이상 된 데이터 자동 정리

### 수동 관리 (월 1회 점검)
- **월 1회 점검**: CLAUDE.md, settings.json, memory 파일에서 더 이상 유효하지 않은 규칙 정리
- **에이전트 점검**: `agents/` 디렉토리 점검 — 에이전트 위임 포인트 적절성, 신뢰도 임계값, 도구 권한, 실제 사용 빈도 확인
- **반복 무시 규칙 분석**: 에이전트가 자주 무시하는 규칙이 있으면 원인 파악 후 표현 수정
- **새 패턴 추가**: 반복해서 나타나는 작업 패턴을 규칙화하여 CLAUDE.md에 반영
- **permissions 갱신**: 프로젝트 변화에 따라 deny/allow 목록 업데이트
- **experience 압축**: 동일 카테고리 5개+ 누적 시 인사이트로 일반화 → 개별 항목 삭제

## 피드백 규칙
- 실수 발견 시: 원인 분석 + 수정안 작성 → **사용자 확인** → 승인 시 memory에 기록
- 기록된 수정안은 이후 프롬프트에 **자동 포함** ("이건 절대 하지 마")
- 거부 시 폐기하거나 피드백 받아 재수정
- 이미 기록된 수정안도 **주기적으로 재검증** — 더 나은 방법 있으면 업데이트 제안

## Reflexion Memory

에러 패턴을 구조화하여 기록, 다음 세션에서 재발 방지.

### 저장 위치
`.claude/memory/reflexion-patterns.jsonl`

### 기록 형식
```jsonl
{"date":"YYYY-MM-DD","pattern":"에러 패턴명","cause":"근본 원인","fix":"적용한 해결책","context":"어떤 프로젝트/상황"}
```

### 기록 트리거
- 에이전트가 같은 에러 2회 연속 반환
- 빌드 실패 후 원인 파악 완료
- 코드 리뷰에서 반복적으로 지적되는 패턴
- 사용자가 "이거 전에도 있었는데"라고 언급

### 활용 방법
1. **세션 시작 시** — `reflexion-patterns.jsonl` 읽기
2. **에이전트 스폰 시** — 관련 패턴이 있으면 프롬프트에 "이 패턴을 피해라"로 포함
3. **월 1회 점검** — 30일+ 된 항목 중 재발 없으면 삭제, 동일 패턴 병합

### 유지 관리
- **최대 50개 항목** — 초과 시 오래된 것부터 삭제
- **재발 시 업데이트** — 같은 패턴이 재발하면 `last_recurred` 날짜 업데이트
- **해결 확정** — 3회 연속 재발 없으면 `resolved: true` 표시
